The dare was one day.

The evening before, over a beer with coworkers, I said I would build the idea in a day. The first commit landed at 06:35. Background replacement worked at 08:00. Virtual try-on followed at 08:23.

Auth, billing, rate limiting, and Sirv storage were in before noon. The first MVP merge is timestamped 12:05. Batch processing and a side-by-side compare mode landed that afternoon. There were 31 commits on day one.

04The production system behind the production system

I built a machine that builds.

AI agents type the code. That is the least interesting part. The leverage comes from giving them bounded work, isolated branches, tests, adversarial reviewers, and a path to converge without turning the repo into soup.

VibeQueue holds the work. The clanker army turns reviewed plans into worker batches. Each batch runs in its own worktree, passes focused tests and quality gates, then gets reviewed by specialist agents before convergence. My job is architecture, specification, judgment, and deciding what deserves to exist.

05The correction built into the record

Velocity created its own problem.

“Construction has outrun proof.”

That line is from the repo’s own July situation assessment. The end-to-end loop exists, but too much of the platform is rollout-gated and too little is backed by named merchants using it every week. The commit graph is evidence of an unusual production system. It is not evidence of product-market fit.

So Q3 has a surface freeze. No new tools, channels, or workflow step types without an explicit decision. The same fleet is moving onto activation, billing boundaries, browser proof, onboarding, App Store quality, and getting ten merchants through the full loop. A build record is stronger when it records the course correction too.
